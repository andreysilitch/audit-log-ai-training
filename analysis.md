# Analysis

## Case

**Продукт:** система для управления производственными процессами в производстве легковых автомобилей.

## Signals

- Feedback signals: L3 — только pre-commit hooks отсутствуют
- Context substrate: L1
- Blast radius control: L1

## Карта доступов

- GitHub: есть
- CI/CD: есть
- Jira: есть
- Staging: есть
- Pre-commit / secrets: частично
   - Pre-commit не используются
   - Vault есть

## Самое слабое место

Проект быстро растёт как с технической стороны, так и со стороны бизнес-функциональности. В команду добавляются новые технологии, фреймворки и новые сценарии использования продукта.

Главная проблема на текущий момент — недостаток актуального контекста о проекте: отсутствуют понятные и поддерживаемые схемы, диаграммы и визуальные модели, которые помогали бы быстро ориентироваться в системе.

## Первый шаг к решению проблемы

Первый реалистичный шаг — попробовать внедрить **автоматическую генерацию схем бизнес-процессов на базе исходного кода**.

Цель — не создать идеальную документацию вручную, а построить процесс, при котором часть схем генерируется автоматически и обновляется вместе с кодом.
